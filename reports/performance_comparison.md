# 레드/화이트 분리 vs 통합 모델 성능 비교

## 1. 요약

| 구분 | Accuracy | F1-macro |
|------|----------|----------|
| **통합 모델** (레드+화이트 한 번에 학습) | **0.4008** (40.1%) | **0.2709** (27.1%) |
| **레드 전용 모델** | **0.8844** (88.4%) | **0.5441** (54.4%) |
| **화이트 전용 모델** | **0.8561** (85.6%) | **0.6513** (65.1%) |
| **분리 모델 가중평균** (테스트 비율 반영) | **0.8631** (86.3%) | **0.6253** (62.5%) |

- **Accuracy**: 통합 40.1% → 분리 가중평균 **86.3%** → **+46.2%p** (약 **2.15배**)
- **F1-macro**: 통합 27.1% → 분리 가중평균 **62.5%** → **+35.4%p** (약 **2.31배**)

---

## 2. 통합 모델 (기존)

- 테스트: 레드+화이트 합쳐서 1,300건, 동일 split 기준 비교.
- **poor** 클래스: precision/recall/f1 모두 0 (거의 예측 불가).
- **normal** / **good**만 어느 정도 구분, 전체적으로 한쪽으로 치우친 예측.

---

## 3. 레드 전용 모델

- 테스트 320건.
- **Accuracy 88.4%**, **F1-macro 54.4%**.
- **normal**: precision 0.90, recall 0.97, f1 0.93.
- **good**: precision 0.76, recall 0.65, f1 0.70.
- **poor**: 레드 테스트셋에 13건뿐이라 여전히 0 (샘플 부족).

---

## 4. 화이트 전용 모델

- 테스트 980건.
- **Accuracy 85.6%**, **F1-macro 65.1%** (레드보다 F1 높음).
- **poor**: precision 0.73, recall 0.22, f1 0.33 → 통합일 때보다 크게 개선.
- **normal**: precision 0.87, recall 0.95, f1 0.91.
- **good**: precision 0.80, recall 0.64, f1 0.71.

---

## 5. 해석

1. **레드와 화이트는 화학 성분 분포가 달라** 한 모델로만 맞추면 결정 경계가 애매해져 성능이 낮았음.
2. **타입별로 나누면** 각 분포에 맞는 경계를 학습해 Accuracy·F1-macro 모두 크게 상승.
3. **가중평균 기준**으로 보면, 같은 테스트 규모(1,300건)에서  
   - Accuracy: 40.1% → 86.3%  
   - F1-macro: 27.1% → 62.5%  
   로, **분리했을 때 성능이 약 2배 이상 좋아진 것**으로 정리할 수 있음.
4. **poor** 클래스는 레드에서는 여전히 어렵고, 화이트에서는 소수 클래스지만 일부 잡을 수 있게 개선됨.

---

## 6. 사용한 결과 파일

- 통합: `reports/evaluation_report.json`
- 레드: `reports/evaluation_red.json`
- 화이트: `reports/evaluation_white.json`

가중평균 식:  
`(320 × red_accuracy + 980 × white_accuracy) / 1300`,  
F1-macro 동일 방식.
